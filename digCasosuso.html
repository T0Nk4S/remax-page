Diagrama de casos de uso

Diagrama de casos de uso de Negocio

Un diagrama de caso de uso de negocio es una representación visual de las interacciones y relaciones entre los diferentes actores (usuarios o sistemas externos) y los procesos de negocio dentro de una organización. Este tipo de diagrama se utiliza para modelar y entender cómo los procesos de negocio satisfacen las necesidades de los actores involucrados.

Casos de Uso

Actor: Usuario

Descripción: Permite que un usuario se registre en el sistema proporcionando datos personales como email, contraseña, nombre, etc.

Relación: Clase Usuario (método registrar()).

Actor: Usuario

Relación: Clase Usuario (método iniciarSesion()).

Actor: Usuario

Relación: Clases Usuario y SolicitudAgente (método enviarSolicitudAgente()).

Descripción: Permite al agente inmobiliario gestionar propiedades (crear, modificar, o eliminar propiedades en el sistema).

Relación: Clases AgenteInmobiliario y Propiedad (método gestionarPropiedad()).

Actor: Agente Inmobiliario

Descripción: Un agente puede asignar una propiedad a un cliente interesado.

Relación: Clase AgenteInmobiliario (método asignarPropiedad()).

CASO DE USO TIPO TEXTO

Actor: Usuario

Descripción: Permite que un usuario se registre en el sistema proporcionando sus datos personales como email, contraseña, nombre, entre otros.

Precondiciones: El usuario no debe estar registrado previamente.

Flujo Principal:

  1. El usuario accede a la opción de registro.
  2. Ingresa sus datos personales (email, contraseña, nombre, etc.).
  3. El sistema valida que el email no esté registrado previamente.
  4. Se guarda la información en el sistema.
  5. El sistema confirma que el registro fue exitoso.

Postcondiciones: El usuario queda registrado en el sistema y puede iniciar sesión.

Actor: Usuario

Descripción: Permite que un usuario inicie sesión en el sistema mediante su email y contraseña.

Precondiciones: El usuario debe estar registrado en el sistema.

Flujo Principal:

  1. El usuario accede a la opción de inicio de sesión.
  2. Ingresa su email y contraseña.
  3. El sistema valida las credenciales ingresadas.
  4. Si las credenciales son correctas, el sistema permite el acceso al usuario.

Postcondiciones: El usuario accede a su cuenta.

Actor: Usuario

Descripción: Permite que un usuario envíe una solicitud para ser considerado como agente inmobiliario.

Precondiciones: El usuario debe estar registrado y no tener el rol de agente inmobiliario.

Flujo Principal:

  1. El usuario accede a la opción "Solicitar ser agente".
  2. Rellena un formulario con datos como su currículum y área de especialidad.
  3. El sistema almacena la solicitud y cambia su estado a "pendiente".
  4. Un administrador revisa y aprueba/rechaza la solicitud.

Postcondiciones: La solicitud queda registrada para ser evaluada.

Actor: Agente Inmobiliario

Descripción: Permite a un agente inmobiliario crear, modificar o eliminar propiedades.

Precondiciones: El agente debe estar registrado y autenticado en el sistema.

Flujo Principal:

  1. El agente accede a la sección de gestión de propiedades.
  2. Selecciona una acción: crear, modificar o eliminar una propiedad.
  3. En caso de creación, ingresa los detalles de la propiedad (dirección, precio, tipo, etc.).
  4. El sistema guarda los cambios y confirma la acción.

Postcondiciones: Las propiedades son actualizadas en el sistema.

Actor: Agente Inmobiliario

Descripción: Un agente puede asignar una propiedad a un cliente interesado.

Precondiciones: El cliente y la propiedad deben estar registrados en el sistema.

Flujo Principal:

  1. El agente accede a la lista de clientes y propiedades.
  2. Selecciona un cliente y una propiedad para asignar.
  3. El sistema registra la asignación y notifica al cliente.

Postcondiciones: La propiedad queda vinculada al cliente.

Actor: Cliente

Descripción: Un cliente consulta las propiedades disponibles según sus preferencias.

Precondiciones: El cliente debe estar registrado y autenticado.

Flujo Principal:

  1. El cliente accede a la opción de búsqueda de propiedades.
  2. Ingresa sus criterios de búsqueda (ubicación, tipo, precio, etc.).
  3. El sistema filtra las propiedades disponibles según los criterios ingresados.
  4. Se muestra una lista de propiedades que cumplen con las preferencias.

Postcondiciones: El cliente visualiza las propiedades disponibles.

Actor: Cliente

Descripción: Permite al cliente realizar una inversión en un proyecto inmobiliario.

Precondiciones: El cliente debe estar registrado y haber seleccionado un proyecto.

Flujo Principal:

  1. El cliente selecciona un proyecto en el que desea invertir.
  2. Ingresa el monto de inversión.
  3. El sistema valida que el monto sea menor o igual al requerido por el proyecto.
  4. Se registra la inversión en el sistema.
  5. El sistema confirma la transacción.

Postcondiciones: El monto invertido queda registrado en el proyecto.

Actor: Cliente, Agente Inmobiliario

Descripción: Incluye acciones como firmar, cancelar o renovar contratos asociados a propiedades.

Precondiciones: Debe existir un contrato previamente registrado.

Flujo Principal:

  1. El cliente o agente accede a la sección de contratos.
  2. Selecciona una acción (firmar, cancelar o renovar).
  3. El sistema actualiza el estado del contrato según la acción realizada.
  4. Se confirma la acción.

Postcondiciones: El estado del contrato es actualizado.

Actor: Cliente

Descripción: Permite al cliente enviar un formulario con preferencias de búsqueda.

Precondiciones: El cliente debe estar registrado y autenticado.

Flujo Principal:

  1. El cliente accede al formulario de consulta.
  2. Completa los campos requeridos (ubicación, tipo de propiedad, etc.).
  3. Envía el formulario.
  4. El sistema almacena los datos y los envía al agente correspondiente.

Postcondiciones: El formulario queda registrado en el sistema.

Actor: Administrador, Agente Inmobiliario

Descripción: Permite cerrar un proyecto una vez cumplido su objetivo de inversión.

Precondiciones: El proyecto debe estar activo y con su monto de inversión completo.

Flujo Principal:

  1. El administrador o agente accede a la lista de proyectos activos.
  2. Selecciona el proyecto que desea cerrar.
  3. El sistema verifica que el monto de inversión esté completo.
  4. Se actualiza el estado del proyecto a "cerrado".

Postcondiciones: El proyecto queda cerrado en el sistema.
